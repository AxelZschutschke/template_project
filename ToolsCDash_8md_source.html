5 # Overview
6 
7 CDash is a dashboard tool developed by KitWare and released as BSD-3 open source tool. Albeit having a little dated
8 looks, it is still a valuable addition to CMake based build chains.
9 
10 How to set up a dockerized cdash / ctest environment:
11 
12 ## Get CDash
13 
14 Clone the [git repository](https://github.com/Kitware/CDash.git) to your local machine and edit the *docker-compose.yml*
15 file to your needs. Especially change the User and Admin accounts and passwords accordingly.
16 
17 Example (probably not fit for production):
18 
19 ~~~~~.yml
20 version: '3.3'
21 services:
22  cdash:
23  image: "kitware/cdash"
24  build: .
25  depends_on:
26  - mysql
27  links:
28  - mysql
29  ports:
30  - "8080:80"
31  environment:
32  CDASH_ROOT_ADMIN_PASS: mysecret
33 
34 
35  CDASH_CONFIG: |
36  $$CDASH_DB_HOST = 'mysql';
37  $$CDASH_DB_NAME = 'cdash';
38  $$CDASH_DB_TYPE = 'mysql';
39  $$CDASH_DB_LOGIN = 'root';
40  $$CDASH_DB_PORT = '';
41  $$CDASH_DB_PASS = '';
42  $$CDASH_DB_CONNECTION_TYPE = 'host';
43 
44  CDASH_STATIC_USERS: |
45  ADMIN root@domain.de mysecret
46 
47  ADMIN axel@domain.de mysecret
48 
49  USER group@clt.de clt
50 
51  mysql:
52  image: "mysql/mysql-server:5.5"
53  environment:
54  MYSQL_ALLOW_EMPTY_PASSWORD: 'yes'
55  MYSQL_ROOT_PASSWORD: ''
56  MYSQL_DATABASE: 'cdash'
57  MYSQL_ROOT_HOST: '%'
58 ~~~~~
59 
60 And build / start the cdash environment via `docker-compose up`
61 
62 This will build the docker environment and starts up an mysql docker and a cdash docker, which is not yet fit for login.
63 During this initial setup, no users have been created. Fix this using `docker-compose run --rm cdash install configure`.
64 
65 This will update the mysql *cdash* database accordingly and add all required tables in the correct version. It will also
66 enter the user accounts as specified in the compose file.

69 ## Set up CMake Project
70 
71 Taken from [cmake.org](https://cmake.org/cmake/help/v3.3/module/CTest.html)
72 
73 To main *CMakeLists.txt*, add the CTest module:
74 
75 ~~~~~.cmake
76 project(MyProject)
77 ...
78 include(CTest)
79 ~~~~~
80 
81 The module automatically creates a *BUILD_TESTING* option that selects whether to enable testing support (ON by
82 default). After including the module, use code like:
83 
84 ~~~~~.cmake
85 if(BUILD_TESTING)
86  # ... CMake code to create tests ...
87 endif()
88 ~~~~~
89 
90 To creating tests when testing is enabled.
91 
92 To enable submissions to a CDash server, create a *CTestConfig.cmake* file at the top of the project with content such
93 as:
94 
95 ~~~~~.cmake
96 set(CTEST_PROJECT_NAME "MyProject")
97 set(CTEST_NIGHTLY_START_TIME "01:00:00 UTC")
98 set(CTEST_DROP_METHOD "http")
99 set(CTEST_DROP_SITE "my.cdash.org")
100 set(CTEST_DROP_LOCATION "/submit.php?project=MyProject")
101 set(CTEST_DROP_SITE_CDASH TRUE)
102 ~~~~~
103 
104 ## Ship Results
105 
106 Use cmake (or ccmake) for configuration of your project. This also enables the ctest environment. For further building
107 testing and submitting, use ctest targets directly, e.g. `ctest -D Experimental`, this will build, test and submit all
108 results automatically. The steps can be carried out manually as well, refer to the ctest manual.
